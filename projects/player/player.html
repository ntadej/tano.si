Fully functional multimedia player. Play SD and HD TV channels.

Tano can play almost every multimedia file, and is primarily meant for SD and HD IP Television channels. You can pause using time shift and set different audio/video settings, which are remembered for each channel.

{% image tano/feature-playlist.jpg class="featurette-image img-circle featurette-left" %}

Channel list is the heart of Tano. Have your channels like on a TV.

Every part of application depends on the channel list: select your channel to watch, browse schedule or plan a recording. Using integrated editor, you can edit and also scan for new channels.

{% image tano/feature-schedule.jpg class="featurette-image img-circle featurette-right" %}

See what is comming up. Use integrated schedule to plan your watching.

Read information about current shows and also schedules for several days in advance. Auto updated from internet using XMLTV format, schedule is also available for browsing independent from playback.

{% image tano/feature-recorder.jpg class="featurette-image img-circle featurette-left" %}

Record your favourite shows. Never miss a movie again.

If you can not watch certain show, you can use integrated recorder to save it for later. Plan with timers and help yourself with schedule. You can set-up daily, weekly and week days timers.

Help & Support

Having trouble watching TV? Hopefully following information will help you.

System requirements

Classic Standard Definition TV channels should play normally on any newer processors, but Hight Definition TV channels need dual-core processor like Intel Core 2 or a graphics card capable of hardware acceleration.

For separate internet and IP TV, two network cards are recommended. One of them should be wired. Connect TV to first, wired network card (eth0). Use second network card, that can be wireless, for internet or ethernet.

Supported OS

Tano runs on Linux, OS X and Windows, other operating systems are not supported. Official packages are available for Ubuntu and Arch Linux.

Configuration

I recommend using a proxy server for watching Multicast IP TV such as UDPXY or Tvheadend. They are easier to configure and will ease your configuration.

If you need two network cards for watching IP TV and browsing internet at the same time, but do not want to use any additional software, you can help yourself with following instructions:

  • Linux
  • OS X
  • Windows
Warning: You might need administrator privileges.

Basic configuration

I recommend you connect TV to eth0 and internet to eth1 or other network connection.

Edit /etc/network/interfaces:

{% highlight vim linenos %} auto lo iface lo inet loopback iface eth0 inet static address 10.*.*.* netmask 255.255.0.0 up route add -net 239.255.0.0 netmask 255.255.0.0 dev eth0 auto eth0 {% endhighlight %}
  • 239.255.0.0 represents IP range from your playlist. Copy and modify this command for every IP range from your playlist.
  • 10.*.*.* represents your network card IP to which TV source is connected

Advanced configuration

Edit /etc/sysctl.conf and make cure the following options are set:

{% highlight vim linenos %} net.ipv4.conf.default.rp_filter=0 net.ipv4.conf.all.rp_filter=0 {% endhighlight %}

If TV still does not work, edit /etc/rc.local to apply sysctl settings at startup. Add the following before exit 0:

{% highlight vim linenos %} sysctl -p {% endhighlight %}

Warning: You might need administrator privileges.

Basic configuration

I recommend your TV source is connected to a network card which says "Local Area Connection" and internet to "Local Area Connection 2". If not obtained automatically, set the static IP address of your network card which is connected to TV.

Information: You can read more information about changing network card IP here.

Advanced configuration

If TV does not work, run this in command line:

{% highlight bash %} route -p add 239.255.0.0 mask 255.255.0.0 10.*.*.* {% endhighlight %}
  • 239.255.0.0 represents IP range from your playlist. Modify this command for every IP range from your playlist.
  • 10.*.*.* represents your network card IP to which TV source is connected
